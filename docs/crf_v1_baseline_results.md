# CRF v1: Baseline эксперименты

**Дата:** 2026-06-17  
**Статус:** Завершено. CRF baseline показал низкий span-F1 (0.07–0.10), что согласуется с BERT baseline.

---

## 1. Цель

Обучить CRF (sklearn-crfsuite) на тех же данных, что и BERT, для сравнения двух семейств моделей. Два эксперимента: CRF-0 (без POS) и CRF-1 (+POS через Stanza).

## 2. Конфигурация

**Данные:**
- Train: train_clean (185 synthetic) + train_noisy (1261 Mathematicon) = 1446 предложений
- Val: 30 предложений (Gosha)
- Test: 50 предложений (Gosha)

**Словарь (построен в runtime):**
- markers.yaml: математические маркеры с ролями (RELATION, OPERATOR, VARIABLE, FUNCTION и т.д.) и уровнями уверенности
- s2l: произношения формул из russian_sentences_train.csv (слова с freq ≥ 50)
- Объединённый словарь: markers + s2l-only (с ролью S2L_MATH)

**Признаки (features):**
- word.lower, suffix[-3:], prefix[:3]
- is_digit, is_latin, is_upper, is_title, word_len
- word_shape (Xxddd-паттерн)
- is_math_word (из объединённого словаря)
- math_role (роль слова: RELATION, OPERATOR, VARIABLE, S2L_MATH, NONE)
- BOS/EOS маркеры
- Контекстное окно ±2 (word.lower, is_math_word, math_role + POS соседей)
- CRF-1 дополнительно: postag (UPOS через Stanza ru)

**Гиперпараметры:** lbfgs, c1=0.1, c2=0.1, max_iterations=200, all_possible_transitions=True

## 4. Анализ

### 4.1. Почему span-F1 низкий

Span-F1 (seqeval strict) требует точного совпадения границ спана. CRF с ручными признаками плохо справляется с определением точных границ по нескольким причинам:

1. **Нет контекстуализированных эмбеддингов.** CRF видит только локальное окно ±2 токена. Для определения границы длинного спана (6-20 токенов) нужен более широкий контекст.

2. **Шумная разметка Mathematicon** (1261 из 1446 тренировочных предложений). Mathematicon — автоматически размеченные данные из другого домена с другим распределением длин спанов (52% одиночных спанов vs 25% у Gosha). CRF учится на шумных паттернах.

3. **Token-F1 vs Span-F1 gap.** Token-F1 ≈ 0.49–0.50 показывает, что CRF частично распознаёт математические зоны, но границы ставит неточно. Тот же паттерн наблюдается у BERT (token-F1 = 0.83, span-F1 = 0.19), просто на другом уровне.

### 4.2. Что даёт POS

POS-теги помогают CRF различать:
- Существительные/прилагательные в математическом контексте (NOUN + is_math_word → более вероятно B-MATH/I-MATH)
- Глаголы и предлоги как маркеры границ спанов (VERB/ADP → вероятнее O)
- Дискурсивные частицы (PART → O)

### 4.3. Сравнение с BERT

| Model | Test Span-F1 | Test Token-F1 |
|-------|-------------|--------------|
| CRF-0 | 0.0738 | 0.4854 |
| CRF-1 | 0.0977 | 0.5047 |
| BERT-0 | 0.1932 | 0.8328 |
| BERT-4 (best) | 0.3287 | 0.8458 |

BERT-0 превосходит лучший CRF (CRF-1) по span-F1 в ~2x, по token-F1 в ~1.65x. Контекстуализированные представления ruBERT дают существенное преимущество на этой задаче.

## 5. Выводы

1. **CRF baseline установлен.** Span-F1 = 0.10 (CRF-1) — отправная точка для сравнения с BERT и для оценки добавления фичей.

2. **POS помогает, но недостаточно.** +2.4 пп — заметный, но не трансформативный прирост.

3. **Основная проблема — данные, не модель.** CRF тренируется на тех же шумных Mathematicon данных. Следующий шаг: обучить CRF на train_clean + train_llm (без Mathematicon) — аналогично тому, что дало прорыв для BERT-4.

4. **CRF остаётся ценным для анализа.** Даже при низком span-F1, CRF даёт интерпретируемые веса признаков, что полезно для понимания задачи и для презентации.
