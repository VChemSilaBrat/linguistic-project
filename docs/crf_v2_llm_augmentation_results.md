# CRF v2: Эксперименты с LLM-сгенерированными данными

**Дата:** 2026-06-17  
**Статус:** Завершено. CRF на LLM-данных (span-F1 = 0.3355) сравнялся с BERT-4 (0.3287). Mathematicon вредит CRF ещё сильнее, чем BERT.

---

## 1. Цель

Проверить, повторяется ли для CRF тот же паттерн, что для BERT: LLM-сгенерированные данные (train_llm) улучшают результат, а шумные данные Mathematicon (train_noisy) вредят.

## 2. Предыстория

CRF v1 (E5) тренировался на train_clean (185) + train_noisy (1261 Mathematicon):
- CRF-0 (без POS): test span-F1 = 0.0738
- CRF-1 (+POS): test span-F1 = 0.0977

Для BERT замена Mathematicon на LLM-данные дала +70% прирост (BERT-0: 0.1932 → BERT-4: 0.3287). Вопрос: будет ли аналогичный эффект для CRF?

## 3. Конфигурация экспериментов

**Словарь:** markers.yaml (641 слово, 11 ролей) + s2l (637 слов, freq≥50) = 1024 слова.

**Признаки:** те же, что в v1 — is_math_word, math_role, word_shape, prefix/suffix, is_digit, is_latin, контекст ±2, BOS/EOS. CRF-3/CRF-5 дополнительно: POS (Stanza ru, 0 misaligned).

**Гиперпараметры:** lbfgs, c1=0.1, c2=0.1, max_iterations=200.

| Эксперимент | Тренировочные данные | Размер | POS |
|------------|---------------------|--------|-----|
| CRF-2 | train_clean + train_llm | 694 | Нет |
| CRF-3 | train_clean + train_llm | 694 | Да |
| CRF-4 | train_clean + train_llm + train_noisy | 1955 | Нет |
| CRF-5 | train_clean + train_llm + train_noisy | 1955 | Да |

## 4. Результаты

### Все эксперименты

| Model | Training Data | Size | POS | Val Span-F1 | Test Span-F1 | Test Token-F1 | Test EM |
|-------|--------------|------|-----|-------------|-------------|--------------|---------|
| CRF-0 (v1) | clean+noisy | 1446 | Нет | 0.0541 | 0.0738 | 0.4854 | 0.0200 |
| CRF-1 (v1) | clean+noisy | 1446 | Да | 0.0769 | 0.0977 | 0.5047 | 0.0200 |
| CRF-2 | clean+llm | 694 | Нет | 0.2444 | 0.3232 | 0.8030 | 0.1200 |
| CRF-3 | clean+llm | 694 | Да | 0.2667 | **0.3355** | 0.8010 | 0.1200 |
| CRF-4 | clean+llm+noisy | 1955 | Нет | 0.1413 | 0.1950 | 0.7492 | 0.0400 |
| CRF-5 | clean+llm+noisy | 1955 | Да | 0.1390 | 0.2012 | 0.7548 | 0.0600 |

### Лучшая модель: CRF-3

- **Test span-F1 = 0.3355** — рост в 3.4x относительно CRF-1 (0.0977)
- **Test token-F1 = 0.8010** — рост в 1.6x относительно CRF-1 (0.5047)
- **Exact match = 12%** — рост в 6x относительно CRF-1 (2%)
- Время обучения: 1.58 сек

## 5. Анализ

### 5.1. LLM-данные дают тот же эффект, что для BERT

| Модель | Без LLM (old) | С LLM (new) | Δ | Рост |
|--------|--------------|-------------|---|------|
| CRF, no POS | 0.0738 | 0.3232 | +0.2494 | +338% |
| CRF, +POS | 0.0977 | 0.3355 | +0.2378 | +243% |
| BERT | 0.1932 | 0.3287 | +0.1355 | +70% |

Для CRF эффект LLM-данных ещё драматичнее, чем для BERT (+338% vs +70%), хотя абсолютные значения сопоставимы. CRF сильнее зависит от качества данных, т.к. не имеет контекстуализированных представлений.

### 5.2. Mathematicon вредит — подтверждение для обеих архитектур

| Сравнение | Без Mathematicon | С Mathematicon | Δ |
|-----------|-----------------|----------------|---|
| CRF, no POS | 0.3232 (CRF-2) | 0.1950 (CRF-4) | **−0.1282** |
| CRF, +POS | 0.3355 (CRF-3) | 0.2012 (CRF-5) | **−0.1343** |
| BERT | 0.3287 (BERT-4) | 0.3072 (BERT-5) | −0.0215 |

Mathematicon вредит CRF в ~6 раз сильнее, чем BERT (−13 пп vs −2 пп). CRF не может «забыть» шумные паттерны — у него нет механизма, аналогичного fine-tuning перезаписи весов в трансформере. Шумные данные напрямую искажают вычисленные переходные вероятности и веса признаков.

### 5.3. CRF ≈ BERT при правильных данных

| Model | Test Span-F1 | Test Token-F1 | Test EM |
|-------|-------------|--------------|---------|
| BERT-4 (best BERT) | 0.3287 | 0.8458 | 0.0867 |
| CRF-3 (best CRF) | 0.3355 | 0.8010 | 0.1200 |
| Δ CRF−BERT | **+0.0068** | −0.0448 | +0.0333 |

CRF-3 слегка превосходит BERT-4 по span-F1 (+0.7 пп) и exact match (+3.3 пп), уступая по token-F1 (−4.5 пп). На данном объёме данных (694 тренировочных предложения) и задаче, преимущество контекстуализированных представлений BERT нивелируется хорошо подобранными ручными фичами (math_role из markers.yaml, is_math_word, word_shape).

### 5.4. POS-теги: умеренный вклад

| Без POS | С POS | Δ |
|---------|-------|---|
| 0.3232 (CRF-2) | 0.3355 (CRF-3) | +0.0123 |
| 0.1950 (CRF-4) | 0.2012 (CRF-5) | +0.0062 |

POS добавляет +1–1.2 пп. Полезно, но не решающе. Основной вклад — от качества данных и math-специфичных фичей.

### 5.5. Топ-20 фичей CRF-3

Наиболее информативные фичи для B-MATH / I-MATH:

1. Контекстные слова-маркеры: `1:word.lower:. (B-MATH)` — точка после токена = начало нового спана
2. Контекстная пунктуация: `1:word.lower:, (I-MATH)` — запятая внутри спана (списки, перечисления формул)
3. Морфологические паттерны: `word[:3]:фик` — префикс «фик» (фиксирую, фиксированная)
4. Лексические триггеры: `word.lower:отличное`, `word.lower:число`, `word.lower:котангенса`

CRF активно использует контекстное окно для определения границ, что объясняет сопоставимый с BERT span-F1.

### 5.6. Анализ ошибок CRF-3

44 из 50 тестовых предложений содержат ошибки. Основные паттерны:

**Splitting длинных спанов** (аналогично BERT):
- Gold: `10 четвертых , 5 вторых плюс 5` → Pred: `10 четвертых` + `5 вторых` + `5` (sent gosha_0007)
- Gold: `90 плюс 30, это вроде как 120` → Pred: `90 плюс 30,` (sent gosha_0008)

**Over-extension на 1 токен:**
- Gold: `тангенса х куб` → Pred: `От тангенса х куб` (sent gosha_0015)
- Gold: `4, 6 делить на 9, 10` → Pred: `Так. Вместо 4, 6 делить на 9, 10` (sent gosha_0010)

**Пропуск коротких спанов:**
- Gold: `Х не равняется 0` → Pred: [] (sent gosha_0018)
- Gold: `альфа-1, альфа-2 коэффициенты, альфа-2` → Pred: `у` (sent gosha_0009)

**Role analysis ошибок:**
- В FN (пропущенных) доминируют токены с ролью NONE (450) — CRF не может распознать math-контекст без словарного подсказки
- В FP (ложных) аналогично NONE (346) — CRF ошибочно размечает нематематические слова вблизи math-маркеров

## 6. Ключевые выводы

1. **LLM-данные — универсально эффективны.** Работают одинаково хорошо для CRF и BERT. Это не артефакт архитектуры — это свойство данных.

2. **Качество данных > архитектура модели.** CRF с 1024-словным словарём и ручными фичами на LLM-данных (0.3355) ≈ fine-tuned ruBERT на тех же данных (0.3287). На малых данных (694 предложения) сложная архитектура не даёт преимущества.

3. **Mathematicon — контрпродуктивен для обеих архитектур.** Добавление 1261 шумного предложения снижает span-F1 на 13 пп для CRF и на 2 пп для BERT. Вывод: при наличии качественных данных, шумные данные только вредят.

4. **CRF обучается за 1.6 секунды** (vs минуты для BERT на GPU). Для итеративного добавления фичей (E6: dep-labels, grammar_match) CRF — идеальная платформа.

## 7. Рекомендации для защиты

1. **Нарратив:** «Мы показали, что качество данных важнее архитектуры модели. CRF с ручными фичами сравнялся с BERT при правильных тренировочных данных. LLM-генерация — ключевой фактор успеха для обеих архитектур.»

2. **Визуализация:** график span-F1 по всем 10 экспериментам — чёткое разделение на "с LLM" (0.32–0.34) и "без LLM" (0.07–0.19).

3. **Практический вывод:** для low-resource NER в специфических доменах CRF + domain-specific features + LLM-generated data — viable и дешёвая альтернатива fine-tuning трансформеров.
