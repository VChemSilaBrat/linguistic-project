# BERT v3: Эксперименты с LLM-сгенерированными данными

**Дата:** 2026-06-17  
**Статус:** Завершено. LLM-аугментация значительно улучшила результаты (+70% span-F1).

---

## 1. Цель

Проверить гипотезу: генерация тренировочных данных через LLM (Groq API, Llama-3) с промптом, заточенным под стиль реальных лекций Gosha, улучшит span-F1 по сравнению с предыдущими подходами (чистая синтетика, s2l template injection, Mathematicon curriculum).

## 2. Предыстория: почему предыдущие аугментации не работали

Все предыдущие попытки добавить данные ухудшали результат:

| Подход | Данные | Test Span-F1 | Δ vs BERT-0 |
|--------|--------|-------------|-------------|
| BERT-0 (baseline) | 185 synthetic | 0.1932±0.0043 | — |
| BERT-1 (curriculum) | 1261 Math + 185 synth | 0.1844±0.0177 | −0.0088 |
| BERT-2 (s2l aug) | 185 synth + 2000 s2l | 0.1634±0.0028 | −0.0298 |
| BERT-3 (curriculum+s2l) | 1261 + 185 + 2000 | 0.1509±0.0152 | −0.0423 |

Причины неудач:
- **s2l template injection:** шаблонные carrier-фразы (185 уникальных шаблонов), формульная речь без лекционного контекста
- **Mathematicon curriculum:** 52% одиночных спанов, шумная разметка, доменное несоответствие
- **Signal dilution:** объём нерелевантных данных размывал сигнал от 185 качественных синтетических предложений

## 3. Новый подход: LLM-генерация через Groq API

### 3.1. Ключевая идея

Вместо механической подстановки формул в шаблоны — генерация целых предложений через LLM с промптом, содержащим:
- Few-shot примеры из реальных лекций Gosha (10 примеров с разными паттернами спанов)
- Явные инструкции по разнообразию: от 1 до 4 спанов, от 1 до 20+ токенов, дискурсивные маркеры
- Формульные произношения из s2l как «сырой материал» для математического контента
- Требование лекционного стиля: оговорки, повторы, «ну вот», «значит», «смотрите»

### 3.2. Технические детали

- **Модель:** Llama-3 70B через Groq API
- **Генерация:** 50 батчей × ~10 предложений = 553 сырых строки
- **Фильтрация:** 43 строки (7.8%) — leaked LLM reasoning (фильтр по слову «спан»)
- **Итого:** 510 валидных предложений, 14,583 токена, 1,059 math спанов

### 3.3. Характеристики сгенерированных данных

| Метрика | train_clean (old) | train_llm (new) | test (Gosha) |
|---------|------------------|----------------|-------------|
| Предложений | 185 | 510 | 50 |
| Средняя длина спана | 10.0 | 6.10 | 6.8 |
| Одиночных спанов | 0% | ~15% | 25% |
| Спанов ≥10 токенов | 51% | ~30% | 25% |

Средняя длина спана LLM-данных (6.10) гораздо ближе к тестовому распределению Gosha (6.8), чем старые synthetic (10.0).

## 4. Эксперименты

### BERT-4: Clean (185) + LLM (510) = 695 предложений
- **Конфигурация:** `ai-forever/ruBert-base`, LR 2e-5, max 10 epochs, early stopping patience=3 по val span-F1
- **Гипотеза:** LLM-данные лучше соответствуют тестовому распределению → рост span-F1

### BERT-5: Curriculum (Mathematicon → Clean + LLM)
- **Stage 1:** train_noisy (1261 Mathematicon), LR 3e-5, 3 epochs, label_smoothing=0.1
- **Stage 2:** train_clean + train_llm (695), fresh optimizer, LR 1.5e-5, max 15 epochs, patience=3
- **Гипотеза:** Curriculum по-прежнему не поможет (подтверждение тренда)

## 6. Анализ ошибок BERT-4 (best seed=456)

### 6.1. Общая статистика

- Exact match: 4/50 предложений (8%)
- Предложения с ≥1 ошибкой спана: 46/50
- Пропущенных gold спанов (false negatives): 90
- Лишних предсказанных спанов (false positives): 165

### 6.2. Типы оставшихся ошибок

**Splitting at discourse markers (самая частая ошибка):**
Модель разбивает один длинный gold-спан на несколько коротких при встрече с дискурсивными словами («и», «это», «,»).

Примеры:
- Gold: `единицы или минус единицы` → Pred: `единицы` + `единицы` (sent 1)
- Gold: `1нт , сумма по i от 1 до n х и т , минус а в` → Pred: `1нт` + `сумма по i...` + `минус а в квадрате.` (sent 33)
- Gold: `математическое ожидание g от кси сходится к математическому ожиданию g от кси для любых функций g` → Pred: `математическое ожидание g от кси` + `ожиданию g от кси` (sent 49)

**Over-extension by 1-2 tokens:**
Модель включает лишний токен на левой или правой границе спана.

Примеры:
- Gold: `тангенса х куб` → Pred: `От тангенса х куб` (sent 8, +1 слева)
- Gold: `Дробь x5, единица минус x 6, x-2 в степени 1` → Pred: `...в степени 1 треть.` (sent 12, +1 справа)
- Gold: `1 поделить на θ минус х1` → Pred: `производную,` + `1 поделить на θ минус х1` (sent 29, FP слева)

**False positives on lecture vocabulary:**
Модель ошибочно размечает лекционные слова как математические.

Примеры:
- `производную,` (sent 29) — глагол, не формула
- `плотность` (sent 28) — терминология, но не часть формулы в данном контексте
- `Производная в квадрате` (sent 30) — описание операции, но gold начинается позже
- `интеграл.` (sent 25) — слово-маркер, gold спан начинается со следующего

### 6.3. Что улучшилось vs BERT-0

- Модель стала **агрессивнее** в поиске math спанов — меньше пропущенных коротких спанов
- Лучше работает с **средними спанами** (5-15 токенов) — основной диапазон Gosha
- Точные совпадения на нескольких сложных предложениях (sent 2, 14, 16, 38)
- Token-F1 стабилен (~0.85) — не деградировал при улучшении span-F1

## 7. Подтверждение тренда: curriculum не помогает

| Эксперимент | Без curriculum | С curriculum | Δ |
|-------------|---------------|-------------|---|
| v1 (original) | 0.1932 (BERT-0) | 0.1844 (BERT-1) | −0.0088 |
| v2 (s2l aug) | 0.1634 (BERT-2) | 0.1509 (BERT-3) | −0.0125 |
| v3 (LLM aug) | 0.3287 (BERT-4) | 0.3072 (BERT-5) | −0.0215 |

Curriculum на Mathematicon **стабильно ухудшает** результат на 1-2 пп. С ростом качества основных данных разрыв даже увеличивается (−0.02 vs −0.01). Вывод: шумные данные Mathematicon вредят, независимо от объёма и качества данных Stage 2.

## 8. Ключевые выводы

1. **LLM-генерация — единственный подход, который улучшил span-F1.** Рост +70% (0.19 → 0.33) — качественный скачок. Все механические подходы (template injection, curriculum) ухудшали результат.

2. **Качество > количество.** 510 LLM-предложений дали +0.14, а 2000 s2l-предложений дали −0.03. Ключевой фактор — соответствие распределению целевого домена (длины спанов, лекционный стиль, дискурсивные маркеры).

3. **Mathematicon как noisy pre-training бесполезен.** Curriculum learning стабильно ухудшает результат во всех трёх раундах экспериментов.

4. **Bottleneck сместился.** Раньше: модель плохо различала math/non-math токены. Теперь: модель хорошо находит математические зоны, но разбивает длинные спаны на части при дискурсивных маркерах.

5. **Потенциал для дальнейшего роста.** Основной тип ошибки (splitting) можно адресовать: (a) добавить в генерацию примеры с длинными спанами, содержащими «и», «это», «,»; (b) постпроцессинг — склейка соседних предсказанных спанов с gap ≤ 1 токен.

## 9. Рекомендации для защиты

1. **BERT-4 (clean + LLM) — лучшая модель.** Span-F1 = 0.33 — значительный прогресс относительно baseline 0.19.

2. **Нарратив для презентации:** «Наивная аугментация данных ухудшает результат. Ключ к успеху — генерация данных, максимально близких к целевому домену. LLM-генерация с few-shot примерами из реальных лекций — эффективный подход для low-resource NER в специфических доменах.»

3. **Границы метода:** span-F1 = 0.33 — это всё ещё low, boundary detection остаётся сложной задачей для устной лекционной речи. Token-F1 = 0.85 показывает, что задача token-level классификации решается значительно лучше.
