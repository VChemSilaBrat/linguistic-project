**Итоговый отчёт по экспериментальной части**  
**Дата:** 2026-06-18  
**Автор:** Павел

## 1. Постановка задачи

Задача — определение спанов (непрерывных последовательностей токенов), содержащих текстовые описания математических формул в русских лекционных транскрипциях. Используется BIO-разметка: B-MATH (начало спана), I-MATH (продолжение), O (не формула).

Пример:

> «Итак, `[B-MATH]` число `[I-MATH]` B `[I-MATH]` называется `[I-MATH]` пределом `[I-MATH]` функций `[I-MATH]` х `[I-MATH]` от `[I-MATH]` двух `[I-MATH]` переменных `[I-MATH]` х `[I-MATH]` и `[I-MATH]` грек `[I-MATH]` в `[I-MATH]` точке `[I-MATH]` A `[O]` .»

Сложность: лекционная речь содержит оговорки, повторы, дискурсивные маркеры («ну вот», «значит», «смотрите»), которые могут встречаться как внутри математических спанов, так и вне них. Границы спанов нечёткие — один и тот же контекст может быть размечен по-разному.

Обучены и сравнены две архитектуры: CRF (sklearn-crfsuite) и ruBERT (ai-forever/ruBert-base, token classification fine-tuning).

**Метрики:**
- **Span-F1** (seqeval, strict mode) — основная метрика. Требует точного совпадения границ спана. Жёсткая, но информативная.
- **Token-F1** (binary) — F1 по отдельным токенам (math vs non-math). Показывает, насколько хорошо модель находит «математические зоны».
- **Exact Match** — доля предложений с полностью правильной разметкой.

- Формульные произношения из s2l (symbol-to-language) как сырой материал
- Требование лекционного стиля: оговорки, повторы, дискурсивные маркеры

Генерация: 50 батчей × ~10 предложений. Фильтрация: 43 из 553 строк (7.8%) — утечки reasoning от LLM (обнаружены по слову «спан» из промпта). Итого: 509 валидных предложений.

### 3.2. Хронология экспериментов и выводы по каждому этапу

**Этап 1: Baselines (BERT-0, CRF-0/1)**

BERT-0 на 185 чистых предложениях: span-F1 = 0.19. CRF-0 на clean+noisy: span-F1 = 0.07. Отправные точки. CRF значительно хуже, потому что тренируется на шумных Mathematicon данных и не имеет контекстуализированных представлений.

**Этап 2: Попытки аугментации — всё ухудшает (BERT-1, 2, 3)**

Три подхода, три неудачи:
- **Curriculum learning** (BERT-1): предобучение на Mathematicon, fine-tuning на clean. Span-F1 упал до 0.18 (−0.01). Шумные данные Mathematicon вредят.
- **S2L template injection** (BERT-2): 2000 предложений, сгенерированных подстановкой формульных произношений в carrier-фразы. Span-F1 упал до 0.16 (−0.03). Шаблонные предложения не похожи на реальные лекции.
- **Curriculum + s2l** (BERT-3): комбинация двух провалившихся подходов. Span-F1 = 0.15 (−0.04). Худший результат.

Вывод: наивная аугментация хуже, чем ничего. Качество данных критичнее их количества.

**Этап 3: LLM-генерация — прорыв (BERT-4, CRF-2/3)**

509 предложений, сгенерированных Llama-3 70B с few-shot промптом из реальных лекций. Результаты:
- BERT-4: span-F1 = 0.33 (+70% к baseline). Первая аугментация, которая помогла.
- CRF-2: span-F1 = 0.32 (+338% к CRF-0). CRF на LLM-данных сразу сравнялся с BERT.
- CRF-3 (+POS): span-F1 = 0.34. POS добавляет +1.2 пп.

Почему сработало: LLM-данные близки к тестовому распределению по длинам спанов (6.1 vs 6.8), лекционному стилю, дискурсивным маркерам.

Повторное подтверждение: **Mathematicon вредит**. CRF-4/5 (с Mathematicon) дают 0.19–0.20, тогда как CRF-2/3 (без него) — 0.32–0.34. Curriculum на BERT-5 (0.31) тоже хуже BERT-4 (0.33).

| | Без Mathematicon | С Mathematicon | Δ |
|---|---|---|---|
| CRF, no POS | 0.3232 | 0.1950 | −0.1282 |
| CRF, +POS | 0.3355 | 0.2012 | −0.1343 |
| BERT | 0.3287 | 0.3072 | −0.0215 |

**Этап 4: Dependency features — финальное улучшение (CRF-6..11)**

Интеграция фичей из rule-based детектора (Stanza dependency parsing + markers.yaml):
- `dep_label` — синтаксическая роль токена в дереве зависимостей
- `dima_bio` — BIO-предсказание rule-based детектора (stacking)

CRF-10 (dep + dima_bio): **span-F1 = 0.3612** — лучший результат проекта.

Неожиданное наблюдение: POS-теги вредят при наличии dima_bio (−1.2..1.4 пп). Rule-based детектор уже использует POS внутренне, повторное добавление создаёт избыточность.

## 4. Анализ ошибок лучшей модели (CRF-10)

43 из 50 тестовых предложений содержат хотя бы одну ошибку. 7 предложений размечены полностью правильно (exact match = 14%).

### 4.1. Splitting длинных спанов — основная ошибка

Модель разбивает один длинный gold-спан на несколько коротких при встрече с дискурсивными маркерами внутри спана.

**Пример 1** (gosha_0001):
```
Gold: «Если для любого эпселен большего нуля, существует такое дельта 
       больше нуля, то для всех точек M , принадлежащих области»
Pred: «существует такое дельта» + «существует такое дельта» + «M»
```
Модель правильно находит математическую зону, но не может удержать непрерывный спан через запятые и союзы.

**Пример 2** (gosha_0007):
```
Gold: «10 четвертых , 5 вторых плюс 5»
Pred: «10 четвертых» + «5 вторых» + «5»
```
Перечисление через запятую разбивается на отдельные спаны.

**Пример 3** (gosha_0002):
```
Gold: «Минус у, деленное на у, у стремится к нулю , но не равно»
Pred: «Минус у, деленное на у, у стремится к нулю»
```
Спан обрезается перед «но не равно» — модель воспринимает «но» как конец формулы.

### 4.2. Over-extension на 1-2 токена

Модель включает лишние токены на границах спана.

**Пример** (gosha_0015):
```
Gold: «тангенса х куб»
Pred: «От тангенса х куб»
```
Предлог «От» ошибочно включён в спан. Аналогичная ошибка у BERT-4 на том же предложении.

**Пример** (gosha_0010):
```
Gold: «альфа»
Pred: «альфа коэффициент я»
```
Модель расширяет однотокенный спан, захватывая контекст.

### 4.3. Полный пропуск спанов

Некоторые спаны не обнаруживаются вообще.

**Пример** (gosha_0018):
```
Gold: «Х не равняется 0»
Pred: [] (ничего не найдено)
```
Предложение «Но только опять же, это условие Х не равняется 0» — модель не распознаёт формулу в контексте с большим количеством нематематических слов.

**Пример** (gosha_0009):
```
Gold: «альфа-1, альфа-2 коэффициенты, альфа-2»
Pred: [] (ничего не найдено)
```
Сложный спан с дефисами и перечислением.

### 4.4. Сравнение ошибок CRF-10 vs BERT-4

Обе модели допускают одинаковые типы ошибок (splitting, over-extension, пропуски), часто на одних и тех же предложениях. Это указывает на то, что bottleneck — в данных и в сложности задачи, а не в архитектуре модели.

CRF-10 чуть лучше по exact match (14% vs 8.7%) и чуть хуже по token-F1 (0.81 vs 0.85). BERT лучше находит отдельные математические токены, CRF точнее ставит границы спанов.

### 4.5. Роли пропущенных и ложных токенов

В пропущенных спанах (false negatives) доминируют токены без маркера в словаре (роль NONE — 53%). Это ожидаемо: модель зависит от словаря маркеров, и токены вне словаря (обычные русские слова внутри формулы) остаются невидимыми для math-специфичных фичей.

В ложных спанах (false positives) аналогичная картина (NONE — 50%). Модель «перебирает» — захватывает нематематические слова рядом с маркерными.

## 5. Ключевые выводы

### 5.1. Качество данных > архитектура модели

CRF с ручными фичами (0.3612) превосходит fine-tuned ruBERT (0.3287) при одних и тех же тренировочных данных. На малом объёме данных (694 предложения) преимущество контекстуализированных представлений BERT не реализуется. Хорошо подобранные domain-specific фичи (math_role, is_math_word, dep_label, dima_bio) компенсируют отсутствие языковой модели.

### 5.2. LLM-генерация — единственный эффективный подход аугментации

Три подхода протестированы: template injection (s2l), curriculum learning (Mathematicon), LLM-генерация. Только LLM-генерация дала положительный результат. Ключевой фактор — соответствие распределению целевого домена: длины спанов, лекционный стиль, дискурсивные маркеры.

| Подход | Предложений | Δ Span-F1 (BERT) | Δ Span-F1 (CRF) |
|--------|-------------|-----------------|-----------------|
| Template injection (s2l) | 2000 | −0.03 | — |
| Curriculum (Mathematicon) | 1261 | −0.01 | — |
| LLM-генерация (Llama-3) | 509 | **+0.14** | **+0.25** |

### 5.3. Шумные данные Mathematicon контрпродуктивны

Во всех экспериментах (6 сравнений) добавление Mathematicon ухудшает результат. Причины: другой домен (письменный vs устный), другое распределение длин спанов (52% одиночных vs 25% у Gosha), шумная автоматическая разметка. CRF страдает от шума сильнее BERT (−13 пп vs −2 пп), потому что не может «перезаписать» шумные паттерны при fine-tuning.

### 5.4. Stacking rule-based и statistical подходов эффективен

Использование BIO-предсказаний rule-based детектора (Dima's formula_bio.py) как фичи для CRF даёт +2.6 пп к span-F1. Два подхода «видят» данные по-разному: rule-based использует деревья зависимостей и словарь маркеров, CRF — статистические паттерны в контексте. Их комбинация превосходит каждый по отдельности.

### 5.5. Span-F1 vs Token-F1 gap

Все модели показывают значительный разрыв: token-F1 ≈ 0.80–0.85, span-F1 ≈ 0.33–0.36. Модели хорошо определяют «математические зоны», но плохо ставят точные границы. Основная причина — дискурсивные маркеры внутри спанов, которые похожи на маркеры конца спана.

## 6. Рекомендации для презентации

### Ключевые слайды

**1. Постановка задачи** — что такое math span, пример с BIO-разметкой, почему это сложно (устная речь, оговорки, нечёткие границы).

**2. Данные** — 4 источника, таблица с размерами. Акцент: gold-стандарт всего 81 предложение → задача low-resource.

**3. Провалившиеся попытки** — s2l injection и curriculum. Важно показать, что наивная аугментация вредит. «Больше данных ≠ лучше».

**4. Прорыв: LLM-генерация** — как устроен промпт, почему сработало (match целевого распределения). Graph: span-F1 до и после, +70% для BERT, +338% для CRF.

**5. CRF ≈ BERT** — при правильных данных интерпретируемая модель не уступает трансформеру. Таблица сравнения. Это неожиданный и сильный результат.

**6. Stacking** — rule-based детектор как фича. Ещё +2.6 пп. Демонстрация комплементарности подходов.

**7. Error analysis** — 2-3 конкретных примера ошибок (splitting, over-extension). Честно показать, где модель ещё ошибается.

**8. Итоговая таблица** — все 18 экспериментов. Визуально выделить лучшую модель (CRF-10: 0.3612).

### Ключевой нарратив

> «В задаче low-resource NER для устной лекционной речи мы исследовали два семейства моделей и несколько стратегий аугментации данных. Наивные подходы (template injection, noisy curriculum) ухудшают результат. LLM-генерация данных с few-shot промптом из реальных лекций — единственный эффективный подход, дающий +70% span-F1 для BERT и +338% для CRF. При правильных данных CRF с domain-specific фичами (span-F1 = 0.36) не уступает fine-tuned ruBERT (0.33), а stacking rule-based детектора даёт дополнительный прирост. Качество данных важнее архитектуры модели.»

### Чего ожидать от вопросов

- «Почему span-F1 такой низкий (0.36)?» — Seqeval strict mode требует точного совпадения границ. Token-F1 = 0.81 показывает, что math-зоны модель находит хорошо. Boundary detection в устной речи — открытая проблема.
- «Почему не попробовали X?» — Фокус был на data-centric approach: не менять модель, а улучшать данные. Это оправдалось (CRF = BERT).
- «Как масштабируется?» — LLM-генерация может генерировать больше данных. CRF обучается за 2 секунды → быстрая итерация.

## Приложение B: Гиперпараметры

**CRF:** sklearn-crfsuite, algorithm=lbfgs, c1=0.1, c2=0.1, max_iterations=200, all_possible_transitions=True.

**BERT:** ai-forever/ruBert-base. BERT-0/4: LR 2e-5, max 10 epochs, early stopping patience=3 по val span-F1. BERT-1/5 (curriculum): Stage 1 — LR 3e-5, 3 epochs, label_smoothing=0.1; Stage 2 — fresh optimizer, LR 1.5e-5, max 15 epochs, patience=3. 3 seed'а (42, 123, 456) для BERT, среднее ± std.
