# Генерация синтетического датасета — Инструкция

## Что здесь лежит

```
prompts/
├── README.md            ← ты здесь
├── batch_01.txt         ← промпт для DeepSeek/Claude
├── batch_02.txt
├── ...
├── batch_19.txt         ← последний батч
├── parse_responses.py   ← парсер ответов → dataset.csv
└── responses/           ← сюда кладёшь ответы (создастся при первом запуске)
    ├── response_01.txt
    ├── response_02.txt
    └── ...
```

## Пайплайн (шаг за шагом)

### 1. Генерация (DeepSeek, ~1 час)

Для каждого `batch_XX.txt`:
1. Открой DeepSeek Chat (chat.deepseek.com) — бесплатно, без лимитов
2. Скопируй **весь текст** из `batch_XX.txt`
3. Вставь в чат, отправь
4. Скопируй ответ целиком
5. Сохрани как `responses/response_XX.txt` (номер совпадает с батчем)

**Совет:** можно открыть несколько вкладок DeepSeek и делать параллельно.

### 2. Парсинг

```bash
python parse_responses.py
```

Результат: `dataset.csv` со столбцами:
- `sentence_clean` — предложение без тегов (чистый текст)
- `sentence_tagged` — предложение с `[MATH]...[/MATH]` разметкой
- `num_formulas` — сколько формул (2 или 3)
- `f1_pron`, `f1_latex` — произношение и LaTeX первой формулы
- `f2_pron`, `f2_latex` — второй
- `f3_pron`, `f3_latex` — третьей (пусто если формул 2)

### 3. Валидация (принеси csv обратно Claude)

Принеси `dataset.csv` в чат — я проверю:
- Валидность LaTeX (через KaTeX/regex)
- Корректность разметки границ
- Статистику по темам
- Найду и починю проблемные строки

## Ожидаемый результат

~185 предложений, каждое с 2-3 формулами = ~200 строк.
19 батчей × ~10 предложений.

## Если модель сломала формат

Не страшно — парсер выведет WARN для кривых строк.
Можно либо перегенерить конкретный батч, либо поправить вручную.
